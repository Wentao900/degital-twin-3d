# 数字孪生仓储系统实施任务分解表

## 1. 文档说明

本文档将当前数字孪生仓储系统改造方案拆分为可执行任务，便于排期、分工、跟踪和验收。

目标：

- 将前端、后端、数据库、视频接入和联动能力拆成独立任务组
- 明确阶段目标、输入依赖、输出产物和验收标准
- 为后续项目管理、甘特图和迭代计划提供基础

## 2. 实施范围

本次实施覆盖以下方向：

- 认证与权限改造
- 现有数字孪生主场景真实数据化
- 数据分析模块新增
- 视频监控模块新增
- 数据库基础表设计与落地
- 报表导出能力
- 告警、视频、场景联动

## 3. 角色建议

建议至少按以下角色协作：

- 产品/业务负责人
- 前端开发
- 后端开发
- 数据库/后端架构
- 视频接入工程师
- 测试

如果团队较小，可由后端兼任数据库设计，前端兼任基础交互设计。

## 4. 阶段划分

建议分为四期：

- 第一期：系统可用化
- 第二期：数据分析中心
- 第三期：视频监控中心
- 第四期：联动与智能化

## 5. 第一期：系统可用化

### 5.1 目标

- 替换当前 mock 登录和菜单
- 补齐现有前端真实依赖接口
- 数字孪生主场景中的轻量状态模块接真实数据
- 告警中心具备基础可用性

### 5.2 任务拆分

#### 任务 A1：认证接口设计与开发

内容：

- 设计登录、登出、当前用户、菜单接口
- 实现 token 鉴权机制

输入依赖：

- `docs/backend-api-draft.md`

输出产物：

- 认证接口
- 鉴权中间件

验收标准：

- 前端可真实登录
- 未登录访问受限页面会跳转登录

#### 任务 A2：菜单与权限模型落地

内容：

- 建立用户、角色、菜单关系
- 按登录用户返回菜单

输出产物：

- 菜单表
- 角色菜单关系
- 菜单接口

验收标准：

- 前端菜单不再依赖本地 mock

#### 任务 A3：告警兼容接口实现

内容：

- 实现当前前端依赖的日/周/月告警和当前告警接口

输出产物：

- 告警查询接口
- 告警日统计表或聚合逻辑

验收标准：

- 当前告警页面可展示真实数据

#### 任务 A4：仓储基础统计接口实现

内容：

- 实现当前驾驶舱依赖的仓储汇总和一周库位趋势接口

输出产物：

- 当前库位汇总接口
- 一周库位统计接口

验收标准：

- 驾驶舱至少一个仓储趋势图使用真实数据

#### 任务 A5：前端登录流程替换

内容：

- 替换 mock 登录
- 请求当前用户与菜单
- 将 token、用户、菜单写入 store

输出产物：

- 登录页改造
- hooks/context 改造
- store 扩展

验收标准：

- 登录成功后可以进入系统
- 刷新页面后仍能恢复登录态

#### 任务 A6：前端请求层增强

内容：

- 请求头注入 token
- 401 统一处理
- 统一错误提示

输出产物：

- `request.ts` 拦截器改造

验收标准：

- 无需手工给每个请求单独写 token 逻辑

#### 任务 A7：数字孪生主场景第一批静态模块替换

内容：

- 替换仓位趋势、告警统计、出入库流水或 AGV 面板中的静态数据

输出产物：

- 至少 2-3 个真实数据面板

验收标准：

- 页面核心数据不再全是 mock

### 5.3 第一期里程碑

里程碑目标：

- 系统可真实登录
- 菜单动态下发
- 当前告警和仓储基础统计可展示

内容：

- 新增分析页面骨架

验收标准：

#### 任务 B2：分析模块 UI 骨架开发

内容：

- 开发仓库分析页
- 开发传感器分析页
- 开发报表页

输出产物：

- 页面骨架
- 基础布局和筛选栏

验收标准：

- 页面结构完整，可接入接口

#### 任务 B3：仓库分析接口开发

内容：

- 仓库汇总
- 仓储趋势
- 库存结构
- 任务统计

输出产物：

- 分析接口
- 聚合查询逻辑

验收标准：

- 仓库分析页图表可展示真实数据

#### 任务 B4：传感器分析接口开发

内容：

- 传感器异常

输出产物：

- 传感器分析接口

验收标准：

- 温湿度等曲线可查询和展示

#### 任务 B5：报表导出机制开发

内容：

- 创建导出任务
- 查询任务状态

输出产物：

- 后台导出任务执行逻辑

验收标准：

- 用户可根据筛选条件导出 Excel/PDF

#### 任务 B6：前端分析图表接入

内容：

- 接入分析接口

输出产物：

- 分析图表页面

验收标准：

- 用户可按仓库、时间查看分析结果

### 6.3 第二期里程碑

里程碑目标：

- 数据分析中心上线
- 支持仓库与传感器分析
- 支持报表导出

## 7. 第三期：视频监控中心

### 7.1 目标

- 新增视频监控入口
- 支持多仓库、多摄像头管理

### 7.2 任务拆分

#### 任务 C1：摄像头数据模型与表结构落地

内容：

- 摄像头台账表
- 流配置表
- 视频事件表

输出产物：

- 数据库表

验收标准：

- 可维护摄像头基础信息和流信息

#### 任务 C2：摄像头管理接口开发

内容：

输出产物：

- 摄像头管理接口

- 可通过接口维护摄像头配置

#### 任务 C3：流媒体接入方案落地

内容：

- 选择媒体网关
- 建立 RTSP 到 WebRTC/HLS 的转流机制
- 建立播放地址签发逻辑

输出产物：

- 流媒体服务
- 播放地址接口

验收标准：

- 浏览器可稳定播放摄像头画面

#### 任务 C4：视频监控页面骨架开发

内容：

- 仓库树
- 摄像头列表
- 视频宫格

输出产物：

- 视频监控首页

验收标准：

- 可查看摄像头列表并选择画面

#### 任务 C5：播放器组件封装

内容：

- 封装 WebRTC/HLS 播放组件
- 处理播放错误和重连

输出产物：

- 通用播放器组件

验收标准：

#### 任务 C6：摄像头事件接口开发

输出产物：

- 视频事件接口

验收标准：

- 摄像头详情页可展示最近事件

### 7.3 第三期里程碑

里程碑目标：

- 视频监控中心上线
- 支持多摄像头列表和播放
- 支持摄像头状态与基础事件查看

## 8. 第四期：联动与智能化

### 8.1 目标

- 打通 3D 场景、告警、视频、分析之间的联动

#### 任务 D1：实时推送通道开发

验收标准：

#### 任务 D2：告警与视频联动

内容：

- 告警可联动到视频监控页

#### 任务 D3：3D 场景与业务实体联动

内容：

输出产物：

- 场景映射表

验收标准：

- 3D 场景点击可跳转或展示业务详情

#### 任务 D4：视频 AI 事件接入

内容：

- 烟雾、入侵、堵塞等事件接入

输出产物：

- 视频事件接入链路

验收标准：

- 视频事件可被查询、展示并联动

#### 任务 D5：预测性分析与健康分析

内容：

- 设备故障趋势
- 传感器异常趋势
- 告警高发区域分析

输出产物：

- 智能分析接口

验收标准：

- 系统不只展示状态，还能展示风险趋势

### 8.3 第四期里程碑

里程碑目标：

- 告警、视频、3D 场景可联动

## 10. 关键依赖关系

### 10.1 前端依赖后端

- 登录改造依赖认证接口
- 菜单改造依赖菜单接口
- 分析页依赖分析接口
- 视频页依赖摄像头和播放地址接口

### 10.2 后端依赖数据库

- 认证接口依赖用户与角色表
- 分析接口依赖基础业务表和历史数据表
- 视频接口依赖摄像头表和流配置表

### 10.3 视频监控依赖外部能力

- 浏览器播放依赖流媒体网关
- 智能分析依赖算法侧或第三方视频 AI 能力

## 11. 风险清单

### 风险 1：前端先写页面，后端字段不稳定

建议：

- 先冻结一期接口结构
- 统一返回字段命名

### 风险 2：视频流直接接浏览器失败

建议：

- 提前确认 `WebRTC/HLS` 技术路径
- 不要以 RTSP 直连作为前端方案

### 风险 3：传感器与设备数据历史量过大

建议：

- 设计初期就考虑分区或时序化

### 风险 4：场景联动对象关系混乱

建议：

- 尽早建立实体与场景对象映射策略

## 12. 验收建议

建议每期至少按以下四类验收：

- 功能验收
- 接口验收
- 数据准确性验收
- 页面联动验收

### 第一期验收重点

- 能登录
- 能看到动态菜单
- 能看到真实告警和基础统计

### 第二期验收重点

- 能做仓库和传感器分析
- 能导出报表

### 第三期验收重点

- 能查看多路摄像头
- 能看到摄像头状态

### 第四期验收重点

- 告警、视频、3D 场景能联动
- 实时状态刷新正常
